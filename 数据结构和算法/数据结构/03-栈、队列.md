# 栈、队列

## 1. 概念

```
栈（Stack）: 后进先出（LIFO）

   入栈(push)          出栈(pop)
      ↓                  ↑
   ┌─────┐            ┌─────┐
   │  30 │ ← 栈顶     │  30 │ ← 先出
   ├─────┤            ├─────┤
   │  20 │            │  20 │
   ├─────┤            ├─────┤
   │  10 │ ← 栈底     │  10 │ ← 后出
   └─────┘            └─────┘

队列（Queue）: 先进先出（FIFO）

   入队(offer)                 出队(poll)
      ↓                          ↑
   ┌─────┬─────┬─────┐        ┌─────┬─────┬─────┐
   │  10 │  20 │  30 │   →→→  │  10 │  20 │  30 │
   └─────┴─────┴─────┘        └─────┴─────┴─────┘
    ↑ 队首(先出)                 队尾(后出) ↑
```

它们都是线性结构上的受限访问模型。

## 2. 来源

栈常用于函数调用栈、表达式求值、括号匹配。  
队列常用于任务调度、消息缓冲、生产者消费者模型。

## 3. 知识点

### 3.1 栈的核心操作

```
栈的操作流程:

初始化: 空栈
  └→ 栈顶指针 = -1

push(10):   push(20):   push(30):   pop():      pop():
┌─────┐     ┌─────┐     ┌─────┐     ┌─────┐     ┌─────┐
│     │     │     │     │  30 │←顶  │     │     │     │
│     │     │     │     ├─────┤     │  20 │←顶  │     │
│     │     │  10 │     │  20 │     ├─────┤     │  10 │←顶
│     │     ├─────┤     ├─────┤     │  10 │     ├─────┤
└─────┘     └─────┘     └─────┘     └─────┘     └─────┘

peek(): 查看栈顶但不移除 → 返回 10
isEmpty(): 检查是否为空 → false
```

### 3.2 队列的核心操作

```
循环队列（解决假溢出）:

普通队列的"假溢出"问题:
[出队]→ [  ] [  ] [  ] [  ] [入队]→  队尾已到数组末尾，但前面有空位！

循环队列解决方案:
   ┌─────┐
   │     │ ← 队尾追上队首表示满
   ├─────┤
   │  30 │
   ├─────┤
←──│  20 │
   ├─────┤
   │  10 │ ← 队首
   └─────┘
      ↑
   数组末尾绕回开头

队空: front == rear
队满: (rear + 1) % capacity == front
```

### 3.3 双端队列（Deque）

```
Deque - 两端都可以操作:

 offerFirst ←┌─────┬─────┬─────┐→ offerLast
             │  10 │  20 │  30 │
 pollFirst  ←└─────┴─────┴─────┘→ pollLast

可以同时模拟栈和队列:
- 只用 offerFirst + pollFirst = 栈
- 只用 offerLast + pollLast = 栈
- offerLast + pollFirst = 队列
```

## 4. 数据结构

示例代码：`src/main/java/org/byron4j/cookbook/algrithms/stackqueue/StackQueueDemo.java`

实现方式：

- 栈：`ArrayDeque`
- 队列：`LinkedList`

```java
// 栈的实现
Deque<Integer> stack = new ArrayDeque<>();
stack.push(10);      // 入栈
int top = stack.pop();  // 出栈

// 队列的实现
Queue<Integer> queue = new LinkedList<>();
queue.offer(10);     // 入队
int head = queue.poll(); // 出队
```

## 5. 最佳实践

1. **Java 中优先用 `ArrayDeque` 替代 `Stack`**。  
2. **队列容量可控场景优先有界队列**。  
3. **并发场景使用 `BlockingQueue` 系列**。  
4. **业务中要明确"满队列/空队列"策略**。  
5. **不要把栈和队列的语义混用**。

### 经典应用

```
括号匹配检测:

输入: "{[()]}"

Step 1: 读 '{' → 入栈 ['{']
Step 2: 读 '[' → 入栈 ['{', '[']
Step 3: 读 '(' → 入栈 ['{', '[', '(']
Step 4: 读 ')' → 匹配 '(' → 出栈 ['{', '[']
Step 5: 读 ']' → 匹配 '[' → 出栈 ['{']
Step 6: 读 '}' → 匹配 '{' → 出栈 []

结果: 栈空，匹配成功！

输入: "{[(])}"

Step 4: 读 ')' → 栈顶是 '('？否！匹配失败！
```
