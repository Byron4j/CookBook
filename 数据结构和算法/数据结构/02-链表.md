# 链表

## 1. 概念

链表是由多个节点通过引用连接而成的线性结构，节点在内存中不要求连续。

```
内存分布对比：

数组（顺序表）:
┌─────┬─────┬─────┬─────┬─────┐
│  10 │  20 │  30 │  40 │  50 │   连续内存
└─────┴─────┴─────┴─────┴─────┘
  地址: 1000  1004  1008  1012  1016

链表:
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ 10│next │───→│ 20│next │───→│ 30│next │───→│ 40│null │   离散内存
└─────────┘    └─────────┘    └─────────┘    └─────────┘
  地址: 2048      1024         4096          512
```

常见类型：

- **单链表**：每个节点仅指向下一个节点
- **双链表**：节点同时保存前驱和后继
- **循环链表**：尾节点再次指向头节点

```
单链表结构:
┌─────┬─────┐    ┌─────┬─────┐    ┌─────┬─────┐
│ data│ next│───→│ data│ next│───→│ data│ null│
└─────┴─────┘    └─────┴─────┘    └─────┴─────┘

双链表结构:
┌─────┬─────┬─────┐    ┌─────┬─────┬─────┐    ┌─────┬─────┬─────┐
│ prev│ data│ next│←──→│ prev│ data│ next│←──→│ prev│ data│ next│
└─────┴─────┴─────┘    └─────┴─────┴─────┘    └─────┴─────┴─────┘

循环链表:
┌─────┬─────┐    ┌─────┬─────┐    ┌─────┬─────┐
│ data│ next│───→│ data│ next│───→│ data│ next│──┐
└─────┴─────┘    └─────┴─────┘    └─────┴─────┘  │
   ↑─────────────────────────────────────────────┘
```

## 2. 来源

链表来源于"动态内存管理 + 频繁插入删除"的需求。  
当数组需要搬移元素时，链表只需要修改引用即可完成局部变更。

## 3. 知识点

### 3.1 时间复杂度对比

| 操作 | 数组 | 单链表 | 双链表 |
|------|------|--------|--------|
| 随机访问 | O(1) | O(n) | O(n) |
| 头部插入 | O(n) | O(1) | O(1) |
| 尾部插入 | O(1)* | O(n) | O(1)** |
| 中间插入 | O(n) | O(n) | O(n) |
| 头部删除 | O(n) | O(1) | O(1) |
| 尾部删除 | O(1) | O(n) | O(1)** |

> * 均摊，可能触发扩容  
> ** 需维护尾指针

### 3.2 头插法 vs 尾插法

```
头插法（逆序）:
初始: 空链表
插入 1: [1] → null
插入 2: [2] → [1] → null
插入 3: [3] → [2] → [1] → null

尾插法（正序）:
初始: 空链表
插入 1: [1] → null
插入 2: [1] → [2] → null
插入 3: [1] → [2] → [3] → null
```

### 3.3 删除操作详解

```
删除中间节点:

Before: [10] → [20] → [30] → [40]
                  ↑
               待删除

Step 1: 找到前驱节点
        [10] → [20] → [30] → [40]
           ↑     ↑
         前驱   当前

Step 2: 修改前驱的 next 指针
        [10] ─────────┐
           ↓          │
          [20] → [30] → [40]
                  ↑
               跳过该节点

Step 3: 释放内存（GC自动处理）
        [10] → [30] → [40]

时间复杂度: O(n) - 需要遍历找到前驱
```

## 4. 数据结构

示例代码：`src/main/java/org/byron4j/cookbook/algrithms/list/SinglyLinkedListDemo.java`

节点定义：
```java
class ListNode {
    int data;           // 数据域
    ListNode next;      // 指针域
}
```

包含能力：

- `addFirst(int value)` - 头插 O(1)
- `addLast(int value)` - 尾插 O(n)
- `removeFirstValue(int value)` - 按值删除 O(n)
- `indexOf(int value)` - 按值查找 O(n)

## 5. 最佳实践

1. **频繁随机访问时不要用链表**。  
2. **需要高频头插/头删可优先链表**。  
3. **修改引用时先保存 `next`**，避免链断裂。  
4. **删除节点时注意头节点特殊分支**。  
5. **通过单测覆盖空链表与单节点场景**。

### 经典应用场景

```
LRU缓存实现:
┌─────────────┐         ┌──────────────────────┐
│   HashMap   │         │     双链表            │
│  (key→Node) │         │  最近使用 ←────→ 最久未使用│
└──────┬──────┘         └──────────────────────┘
       │
       ↓
   key查找 → O(1)
       │
       ↓
   移动到链表头部 → O(1)
       │
       ↓
   淘汰尾部节点 → O(1)
```
