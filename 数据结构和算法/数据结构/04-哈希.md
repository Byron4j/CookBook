# 哈希

## 1. 概念

哈希表通过哈希函数把键映射到数组下标，实现近似 `O(1)` 的查询与插入。

```
哈希表核心结构:

键(key) ──→ 哈希函数 ──→ 索引(index) ──→ 数组[index]
  │                              │
  │                              ↓
  │                         ┌─────────┐
  │                         │ 桶数组   │
  │                         ├────┬────┼────┬────┬────┐
  └────────────────────────→│ 0  │ 1  │ 2  │ 3  │ 4  │
                            ├────┼────┼────┼────┼────┤
                            │    │ A  │    │ B  │ C  │
                            └────┴────┴────┴────┴────┘

冲突示例:
hash("Alice") = 1  →  数组[1] = Alice
hash("Bob") = 3    →  数组[3] = Bob
hash("Charlie") = 4 → 数组[4] = Charlie
hash("David") = 1   →  数组[1] 已被占用！冲突！
```

核心组成：

- **哈希函数**：将 key 转换为数组索引
- **桶数组**：存储数据的底层数组
- **冲突解决策略**：拉链法 / 开放寻址

## 2. 来源

哈希结构来源于"以空间换时间"的工程需求：  
当数据规模增长时，希望避免顺序扫描，直接定位数据所在桶。

## 3. 知识点

### 3.1 哈希函数设计

```
好的哈希函数特征:

均匀分布              差劲的哈希函数
┌───┬───┬───┐        ┌───┬───┬───┐
│ A │ B │ C │        │   │ABC│   │
├───┼───┼───┤        ├───┼───┼───┤
│ D │ E │ F │        │   │DEF│   │
├───┼───┼───┤        ├───┼───┼───┤
│ G │ H │ I │        │   │GHI│   │
└───┴───┴───┘        └───┴───┴───┘
 均匀分散              聚集到少数桶
```

### 3.2 冲突解决策略

**拉链法（Separate Chaining）:**
```
数组 + 链表:

┌─────┬─────┬─────┬─────┬─────┐
│  0  │  1  │  2  │  3  │  4  │
├─────┼─────┼─────┼─────┼─────┤
│     │ A→D→│     │ B→  │ C→  │
│     │  G  │     │     │     │
└─────┴─────┴─────┴─────┴─────┘

A, D, G 都映射到索引 1，形成链表
查询时先找到桶，再遍历链表
```

**开放寻址（Open Addressing）:**
```
线性探测示例:

插入 A (hash=1):    插入 B (hash=3):    插入 C (hash=1):
┌───┬───┬───┐       ┌───┬───┬───┐       ┌───┬───┬───┐
│   │ A │   │       │   │ A │   │       │   │ A │ C │
└───┴───┴───┘       ├───┼───┼───┤       ├───┼───┼───┤
                    │   │   │ B │       │   │   │ B │
                    └───┴───┴───┘       └───┴───┴───┘

C 应该放在索引 1，但已被 A 占用
线性探测: 1→2(空)→放入 C
```

### 3.3 负载因子与扩容

```
负载因子 = 元素个数 / 桶数量

┌────────────────────────────────────┐
│ 负载因子 < 0.75: 性能良好           │
├────────────────────────────────────┤
│ 负载因子 > 0.75: 冲突增加，考虑扩容  │
├────────────────────────────────────┤
│ 扩容流程:                           │
│ 1. 创建更大的数组（通常 2 倍）       │
│ 2. 重新计算所有元素的 hash          │
│ 3. 放入新数组（rehash）             │
└────────────────────────────────────┘
```

## 4. 数据结构

示例代码：`src/main/java/org/byron4j/cookbook/algrithms/hash/SimpleHashTableDemo.java`

示例采用：

- 数组 + 单链表（拉链法）
- `put/get/keys` 三个核心操作

```java
class HashNode {
    String key;
    Object value;
    HashNode next;  // 冲突时链向下一个节点
}

class HashTable {
    HashNode[] buckets;  // 桶数组
    int capacity = 16;   // 初始容量
    int size = 0;        // 元素个数
    
    int hash(String key) {
        return key.hashCode() % capacity;
    }
}
```

## 5. 最佳实践

1. **key 类型要有稳定且分布较好的 `hashCode`**。  
2. **关注负载因子，避免长链退化**。  
3. **对安全敏感场景考虑哈希碰撞攻击**。  
4. **并发写场景使用 `ConcurrentHashMap`**。  
5. **统计冲突率用于容量调优**。

### Java 中的哈希实现

```
HashMap vs ConcurrentHashMap:

HashMap (单线程):
┌─────────────────────────────┐
│ 数组 + 链表/红黑树            │
│ 非线程安全                   │
│ 允许 null key/value          │
└─────────────────────────────┘

ConcurrentHashMap (并发):
┌─────────────────────────────┐
│ 分段锁（JDK7）/ CAS（JDK8）   │
│ 线程安全                     │
│ 不允许 null key/value        │
│ 读操作无锁                   │
└─────────────────────────────┘
```
