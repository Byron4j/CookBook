# 红黑树

## 1. 概念

红黑树是一种自平衡二叉查找树，通过颜色约束确保最长路径不会超过最短路径的两倍。

```
红黑树示例（节点带颜色标记）:

            [30B]           ← 根节点（黑色）
           /     \
        [20B]   [40B]
        /   \      \
     [10B] [25R]  [50R]
                    \
                   [60B]

图例: B=黑色, R=红色
```

关键性质（简化）：

- 节点颜色非红即黑
- 根节点为黑
- 红节点不能有红孩子（红色不相邻）
- 任意节点到叶子空节点路径上的黑节点数相同（黑高相同）

## 2. 来源

红黑树为了解决 BST 退化问题而生，目标是在较少旋转成本下保证近似平衡。  
在工程中被广泛用于有序映射结构。

## 3. 知识点

### 3.1 红黑树 vs AVL树

```
比较维度:

                    红黑树              AVL树
平衡条件:      较宽松(黑高相同)      较严格(高度差≤1)
树高上限:      ≤ 2log₂(n+1)          ≤ 1.44log₂(n+2)
查询性能:      O(log n)              O(log n)（更快）
插入旋转:      最多2次               最多1次
删除旋转:      最多3次               最多 O(log n) 次
适用场景:      插入删除频繁          查询频繁

结论:
- 写多读少 → 红黑树
- 读多写少 → AVL树
```

### 3.2 插入修复流程

```
插入新节点（默认红色）→ 可能破坏性质 → 需要修复

Case 1: 树为空
    新节点设为黑色（根节点必须是黑色）
    
Case 2: 父节点是黑色
    无需修复，直接插入
    
Case 3: 父节点是红色（叔叔也是红色）
    父节点变黑
    叔叔变黑
    祖父变红
    以祖父为当前节点继续向上修复
    
Case 4: 父节点是红色，叔叔是黑色（需要旋转）
    Subcase 4a: 当前节点是右孩子，父节点是左孩子
        对父节点左旋 → 变成 Subcase 4b
        
    Subcase 4b: 当前节点是左孩子，父节点是左孩子
        父节点变黑
        祖父变红
        对祖父右旋
        
    Subcase 4c: 当前节点是左孩子，父节点是右孩子
        对父节点右旋 → 变成 Subcase 4d
        
    Subcase 4d: 当前节点是右孩子，父节点是右孩子
        父节点变黑
        祖父变红
        对祖父左旋
```

**插入修复图解:**
```
Case 3 示例（颜色翻转）:

Before:            After:
       [10B]              [10R]      ← 祖父变红
      /    \             /    \
   [5R]   [15R]       [5B]   [15B]  ← 父叔变黑
   /  \    /  \       /  \    /  \
 [2R]              [2R]
 
新节点 2 插入后，父节点 5 和叔叔 15 都是红色
进行颜色翻转，然后继续检查祖父 10 的父节点


Case 4b 示例（左旋+变色）:

Before:            After:
       [10B]              [5B]
      /    \             /    \
   [5R]   [20B]      [2R]   [10R]
   /  \              /      /    \
 [2R]              [1]    [7]    [20B]
 /
[1]

新节点 1 插入后，形成左-左结构
对祖父 10 右旋，然后变色
```

### 3.3 删除修复

```
删除黑色节点后，黑高被破坏，需要修复:

Case 1: 兄弟是红色
    兄弟变黑，父节点变红
    对父节点旋转（左或右）
    重新指定兄弟
    
Case 2: 兄弟是黑色，且兄弟的两个孩子都是黑色
    兄弟变红
    向上移动到父节点
    
Case 3: 兄弟是黑色，兄弟的远侄子是红色
    兄弟颜色 = 父节点颜色
    父节点变黑
    远侄子变黑
    对父节点旋转
    结束
    
Case 4: 兄弟是黑色，兄弟的近侄子是红色，远侄子是黑色
    近侄子变黑
    兄弟变红
    对兄弟旋转
    重新指定兄弟 → 变成 Case 3
```

## 4. 数据结构

示例代码：`src/main/java/org/byron4j/cookbook/algrithms/tree/TreeMapRedBlackTreeDemo.java`

说明：

- Java `TreeMap` 底层即红黑树
- 示例演示有序遍历、边界查询与邻近查询

```java
// TreeMap 红黑树的使用
TreeMap<Integer, String> map = new TreeMap<>();
map.put(50, "A");
map.put(30, "B");
map.put(70, "C");
map.put(20, "D");
map.put(40, "E");

// 有序遍历
for (Integer key : map.keySet()) {
    System.out.println(key + "=" + map.get(key));  // 20,30,40,50,70
}

// 边界查询
System.out.println(map.firstKey());  // 20
System.out.println(map.lastKey());   // 70
System.out.println(map.floorKey(45)); // 40 (≤45的最大值)
System.out.println(map.ceilingKey(45)); // 50 (≥45的最小值)
```

## 5. 最佳实践

1. **业务优先直接使用 `TreeMap`/`TreeSet`**。  
2. **手写红黑树前先确认是否真的需要**。  
3. **调试时记录颜色和旋转步骤**。  
4. **并发读写不要直接用 `TreeMap`**。  
5. **大规模写多读少可评估跳表替代**。

### 红黑树的应用场景

```
Java 中的红黑树应用:

┌─────────────────────────────────────┐
│ java.util.TreeMap                    │
│ - 基于红黑树的有序映射                │
│ - 按键的自然顺序或比较器排序           │
├─────────────────────────────────────┤
│ java.util.TreeSet                    │
│ - 基于红黑树的有序集合                │
│ - 无重复元素，有序存储                │
├─────────────────────────────────────┤
│ java.util.HashMap (JDK8+)           │
│ - 链表长度 > 8 时转为红黑树           │
│ - 提升查找性能从 O(n) 到 O(log n)     │
└─────────────────────────────────────┘
```
