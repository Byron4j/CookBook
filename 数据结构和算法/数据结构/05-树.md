# 树

## 1. 概念

树（Tree）是典型的非线性结构，用于表达"层级关系"。  
和线性表不同，树中的一个节点可以有多个后继节点。

```
树的基本结构:

            [A]           ← 根节点(Root)
           / | \
         [B][C][D]       ← 子节点(Children)
        / \    / \
      [E][F] [G][H]      ← 叶子节点(Leaf)
      
术语说明:
- A 是 B, C, D 的父节点(Parent)
- B, C, D 是 A 的子节点(Children)
- E, F, G, H 没有子节点，称为叶子节点(Leaf)
- 节点 A 的度 = 3 (子节点个数)
- 树的度 = 3 (所有节点度数的最大值)
- 节点 E 的深度 = 2 (从根到该节点的边数)
- 节点 A 的高度 = 2 (从该节点到最深叶子的边数)
- 树的高度 = 2
```

在工程上，最常见的是二叉树（每个节点最多两个子节点）。

## 2. 来源

树结构来源于现实中的分层关系抽象，例如：

- 组织架构图
- 文件目录
- 家族谱系
- 编译器语法树

只要对象之间具有"父子层级"而非"单链顺序"，就更适合用树建模。

## 3. 知识点

### 3.1 二叉树分类

```
满二叉树 (Full Binary Tree):
            [1]
           /   \
         [2]   [3]
        / \    / \
      [4][5] [6][7]
      
所有层都被填满，叶子节点都在最底层

完全二叉树 (Complete Binary Tree):
            [1]
           /   \
         [2]   [3]
        / \    /
      [4][5] [6]
      
除了最后一层，其他层都满，最后一层从左到右填充

二叉搜索树 (BST):
            [5]
           /   \
         [3]   [8]
        / \    / \
      [1][4] [7][9]
      
左子树 < 根 < 右子树

平衡二叉树:
            [4]
           /   \
         [2]   [6]
        / \    / \
      [1][3] [5][7]
      
任意节点左右子树高度差 ≤ 1
```

### 3.2 遍历方式

```
示例二叉树:
            [1]
           /   \
         [2]   [5]
        /     / \
      [3]   [6] [7]
       \
       [4]

前序遍历 (根-左-右):  1 → 2 → 3 → 4 → 5 → 6 → 7
中序遍历 (左-根-右):  3 → 4 → 2 → 1 → 6 → 5 → 7
后序遍历 (左-右-根):  4 → 3 → 2 → 6 → 7 → 5 → 1
层序遍历 (逐层):      1 → 2 → 5 → 3 → 6 → 7 → 4
```

**遍历流程图:**
```
前序遍历递归过程:

preOrder(node):
    if node == null: return
    visit(node)           ← 先访问根
    preOrder(node.left)   ← 再遍历左子树
    preOrder(node.right)  ← 最后遍历右子树
    
执行顺序可视化:
            [1]✓
           /   \
         [2]✓  [5]✓
        /      / \
      [3]✓   [6]✓[7]✓
       \
       [4]✓

访问顺序: 1 → 2 → 3 → 4 → 5 → 6 → 7
```

### 3.3 存储方式

```
链式存储（通用）:

┌─────────┐
│  data   │
├─────────┤
│  left   │──→ ┌─────────┐
├─────────┤    │  data   │
│  right  │──→ ├─────────┤
└─────────┘    │  left   │──→ ...
               ├─────────┤
               │  right  │──→ ...
               └─────────┘

顺序存储（适合完全二叉树）:

数组索引:  0    1    2    3    4    5    6
         [A]  [B]  [C]  [D]  [E]  [F]  [G]
         
节点关系:
- 节点 i 的左子节点 = 2i + 1
- 节点 i 的右子节点 = 2i + 2
- 节点 i 的父节点 = (i-1) / 2

例如: 节点 B(索引1) 的左子节点 = 2*1+1 = 3 → D
```

## 5. 最佳实践

1. **构造与遍历分离**：构造树只负责建模，遍历负责输出/聚合。  
2. **遍历优先返回集合**，避免直接打印影响复用和测试。  
3. **输入序列约定要明确**（前序 or 层序），否则结果不可预期。  
4. **对 `null` 和空输入做防御式处理**。  
5. **深度可能很深时，评估递归栈风险并考虑迭代版**。

### 树的经典应用

```
表达式树:

      [*]

   [+]   [C]
  /   \
[A]   [B]

中序遍历: A + B * C
后序遍历: A B + C * (后缀表达式)
```
