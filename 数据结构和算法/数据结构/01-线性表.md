# 线性表

## 1. 概念

线性表（Linear List）是最基础的数据结构之一。它的核心特征是：  
除了首尾元素，其他每个元素都只有唯一前驱和唯一后继，逻辑上呈"一条线"。

```
逻辑结构示意图：

  前驱    前驱    前驱           后继    后继
    ↓       ↓       ↓             ↓       ↓
   [a₁] → [a₂] → [a₃] → ... → [aₙ₋₁] → [aₙ]
     ↑                                      ↑
   首元素(无前驱)                        尾元素(无后继)
```

常见实现有两类：

- **顺序表**（数组实现）
- **链表**（指针/引用实现）

本文优先讲顺序表，并给出可运行代码。

## 2. 来源

线性表的思想来自"顺序组织与逐项处理"的通用场景，例如：

- 成绩列表按下标访问
- 消息队列中的线性消费
- SQL 查询结果集按行迭代

在计算机早期内存模型中，连续内存块是最高效的访问方式之一，因此数组化线性表成为很多高级数据结构（栈、队列、堆）的基础。

## 3. 知识点

### 3.1 核心特性

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 按下标访问 | O(1) | 直接通过索引计算地址 |
| 尾部插入 | O(1) | 均摊，可能触发扩容 |
| 中间插入 | O(n) | 需要搬移后续元素 |
| 尾部删除 | O(1) | 直接移除最后一个元素 |
| 中间删除 | O(n) | 需要搬移后续元素 |
| 按值查找 | O(n) | 需要顺序扫描 |

### 3.2 扩容策略

```
扩容流程：
┌─────────────┐
│  插入元素    │
└──────┬──────┘
       ↓
┌─────────────┐     是
│ size == capacity? │──→┌─────────────┐
└─────────────┘         │ 创建新数组    │
       │ 否             │ (通常1.5~2倍) │
       ↓                └──────┬──────┘
┌─────────────┐                ↓
│ 直接放入末尾  │         ┌─────────────┐
└─────────────┘         │ 拷贝旧数据    │
                        └──────┬──────┘
                               ↓
                        ┌─────────────┐
                        │ 放入新元素    │
                        └──────┬──────┘
                               ↓
                        ┌─────────────┐
                        │ size++      │
                        └─────────────┘
```

## 4. 数据结构

### 4.1 顺序表结构

```
内存布局：
┌─────────────────────────────────────────────────────────┐
│  数组引用                                               │
│    ↓                                                    │
│  ┌─────┬─────┬─────┬─────┬─────┬─────┬─────┐            │
│  │  10 │  20 │  30 │  40 │  50 │     │     │  ← 底层数组 │
│  └─────┴─────┴─────┴─────┴─────┴─────┴─────┘            │
│    ↑    ↑    ↑    ↑    ↑                                │
│   [0]  [1]  [2]  [3]  [4]                               │
│                                                         │
│  size = 5    capacity = 7                               │
└─────────────────────────────────────────────────────────┘
```

顺序表最小模型：

- `elements`: 底层数组
- `size`: 当前元素个数

- `src/main/java/org/byron4j/cookbook/algrithms/list/LinearListDemo.java`

包含以下可运行操作：

- 尾部新增 `add(value)`
- 指定位置插入 `add(index, value)`

- 按值查找 `indexOf(value)`
- 导出数组 `toArray()`

### 4.2 插入操作流程

```
在索引 2 处插入元素 25：

原始: [10, 20, 30, 40, 50]
            ↑
          插入点

Step 1: 元素后移
       [10, 20,  _, 30, 40, 50]
                 ↑
               腾出位置

Step 2: 放入新元素
       [10, 20, 25, 30, 40, 50]
                 ↑
               新元素

时间复杂度: O(n) - 需要移动 n-index 个元素
```

## 5. 最佳实践

1. **预估容量，降低扩容次数**。  
2. **对外暴露只读视图或副本**，避免内部数组泄漏。  
3. **高频删除场景避免顺序表**，优先链表或双端队列。  
4. **插入前统一做边界检查**，避免隐藏 bug。  
5. **配套测试覆盖边界值**（空表、越界、扩容临界点）。

### 适用场景对比

```
                    读多写少
                       ↑
                       │
    频繁随机访问 ←─────┼─────→ 需要顺序访问
                       │
                       ↓
                    写多读多
    
    读多写少 + 随机访问: 顺序表（数组）✓
    写多读多 + 频繁插入删除: 链表 ✓
```
