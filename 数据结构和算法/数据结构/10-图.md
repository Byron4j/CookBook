# 图

## 1. 概念

图（Graph）是由顶点（Vertex）和边（Edge）组成的非线性数据结构，用于表示多对多的关系。

```
无向图示例:

    [A]────[B]
    │ ╲    │
    │   ╲  │
    [C]────[D]

边: (A,B), (A,C), (A,D), (B,C), (B,D), (C,D)
每个边没有方向

有向图示例:

    [A]───→[B]
    │ ╲    ↑
    │   ╲  │
    ↓     ╲│
    [C]───→[D]

边: <A,B>, <A,C>, <A,D>, <C,D>, <D,B>
每个边有方向

带权图示例:

        4
    [A]────[B]
    │      │
   2│   1  │3
    │      │
    [C]────[D]
        5

边带权重，表示距离、成本等
```

## 2. 来源

图结构来源于现实世界的网状关系：

- **社交网络**：用户是顶点，好友关系是边
- **地图导航**：地点是顶点，道路是边（带权）
- **项目管理**：任务是顶点，依赖关系是边
- **计算机网络**：设备是顶点，连接是边

## 3. 知识点

### 3.1 图的表示方法

**邻接矩阵（Adjacency Matrix）:**
```
        A   B   C   D
      ┌───┬───┬───┬───┐
    A │ 0 │ 1 │ 1 │ 1 │    1 表示有边，0 表示无边
      ├───┼───┼───┼───┤
    B │ 1 │ 0 │ 1 │ 1 │    无向图: 矩阵对称
      ├───┼───┼───┼───┤
    C │ 1 │ 1 │ 0 │ 1 │
      ├───┼───┼───┼───┤
    D │ 1 │ 1 │ 1 │ 0 │
      └───┴───┴───┴───┘

空间复杂度: O(V²)
适合: 稠密图（边数接近 V²）
查询两点是否有边: O(1)
```

**邻接表（Adjacency List）:**
```
A → [B, C, D]
B → [A, C, D]
C → [A, B, D]
D → [A, B, C]

空间复杂度: O(V + E)
适合: 稀疏图（边数远小于 V²）
查询某点的所有邻居: O(degree)
```

**对比:**
```
                邻接矩阵        邻接表
空间复杂度:      O(V²)          O(V+E)
添加边:          O(1)           O(1)
删除边:          O(1)           O(E)
查询边:          O(1)           O(V)
遍历邻居:        O(V)           O(degree)
适用场景:        稠密图          稀疏图

实际应用:
- 社交网络（稀疏）: 邻接表
- 完全图（稠密）: 邻接矩阵
```

### 3.2 图的遍历

**深度优先搜索（DFS）:**
```
DFS 遍历过程:

图结构:          遍历顺序:

    [A]           1. 访问 A
    │ \           2. 访问 B（A的邻居）
    │  \          3. 访问 D（B的邻居）
   [B]  [C]       4. D 的邻居都已访问，回溯到 B
   │ \            5. B 的邻居都已访问，回溯到 A
   │  \           6. 访问 C（A的另一个邻居）
  [D]  [E]        7. 访问 E（C的邻居）
                  8. 完成

结果: A → B → D → C → E

算法（递归）:
DFS(node):
    mark node as visited
    for each neighbor of node:
        if neighbor not visited:
            DFS(neighbor)

算法（迭代）:
DFS(node):
    stack.push(node)
    while stack not empty:
        current = stack.pop()
        if current not visited:
            mark visited
            for each neighbor of current:
                stack.push(neighbor)
```

**广度优先搜索（BFS）:**
```
BFS 遍历过程:

图结构:          遍历顺序:

    [A]           1. 访问 A，入队 [A]
    │ \           2. A 出队，邻居 B,C 入队 [B, C]
    │  \          3. B 出队，邻居 D 入队 [C, D]
   [B]  [C]       4. C 出队，邻居 E 入队 [D, E]
   │ \            5. D 出队，无新邻居 [E]
   │  \           6. E 出队，完成
  [D]  [E]
                  结果: A → B → C → D → E

算法:
BFS(node):
    queue.enqueue(node)
    mark node as visited
    while queue not empty:
        current = queue.dequeue()
        for each neighbor of current:
            if neighbor not visited:
                mark visited
                queue.enqueue(neighbor)
```

### 3.3 最短路径算法

**Dijkstra 算法（单源最短路径，无权/正权图）:**
```
算法流程:

初始化:
- dist[A] = 0, dist[其他] = ∞
- 优先队列: [(0, A)]

Step 1: 取出 A(距离0)
        更新邻居: dist[B]=4, dist[C]=2
        队列: [(2, C), (4, B)]

Step 2: 取出 C(距离2)
        更新邻居: dist[D]=5 (2+3), dist[E]=7 (2+5)
        队列: [(4, B), (5, D), (7, E)]

Step 3: 取出 B(距离4)
        更新邻居: dist[D] 保持 5 (4+3 > 5)
        队列: [(5, D), (7, E)]

Step 4: 取出 D(距离5)
        更新邻居: dist[E]=6 (5+1)
        队列: [(6, E)]

Step 5: 取出 E(距离6)
        完成

时间复杂度: O((V+E)logV)
```

## 4. 数据结构

图的常见存储实现：

```java
// 邻接表实现
class Graph {
    int V;  // 顶点数
    List<Integer>[] adj;  // 邻接表
    
    Graph(int V) {
        this.V = V;
        adj = new List[V];
        for (int i = 0; i < V; i++)
            adj[i] = new ArrayList<>();
    }
    
    void addEdge(int u, int v) {
        adj[u].add(v);  // 有向图
        // adj[v].add(u);  // 无向图需要反向边
    }
}

// 带权图
class WeightedGraph {
    class Edge {
        int to;
        int weight;
        Edge(int to, int weight) {
            this.to = to;
            this.weight = weight;
        }
    }
    
    List<Edge>[] adj;
    // ...
}
```

## 5. 最佳实践

1. **稀疏图优先使用邻接表**，空间效率更高。  
2. **稠密图或需要快速查询边时**，使用邻接矩阵。  
3. **DFS 适合路径搜索、连通性判断**。  
4. **BFS 适合最短路径（无权图）、层级遍历**。  
5. **带权图最短路径使用 Dijkstra（正权）或 Bellman-Ford（负权）**。

### 图的应用场景

```
社交网络分析:

用户A ──好友── 用户B
  │              │
  └──好友── 用户C

- 共同好友: A 和 B 的共同邻居
- 最短路径: A 到 D 的最短好友链
- 社群发现: 强连通分量

网络路由:

路由器A ──1ms── 路由器B
   │              │
  5ms           2ms
   │              │
路由器C ──3ms── 路由器D

- 最短路径: Dijkstra 算法
- 负载均衡: 最大流算法

项目管理（拓扑排序）:

任务A ──→ 任务B ──→ 任务D
   │         │
   └────→ 任务C ──→ 任务E

- 依赖关系: 有向边
- 执行顺序: 拓扑排序
- 关键路径: 最长路径
```
