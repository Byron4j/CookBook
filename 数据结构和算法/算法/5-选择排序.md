# 选择排序算法

## 1. 概念

选择排序（Selection Sort）是一种简单直观的比较排序算法，每次从未排序部分选择最小（或最大）元素放到已排序部分的末尾。

- 平均时间复杂度：`O(n²)`
- 最坏时间复杂度：`O(n²)`
- 最好时间复杂度：`O(n²)`
- 空间复杂度：`O(1)`（原地排序）
- 稳定性：**不稳定排序**

## 2. 来源

选择排序思想直观：在乱序的牌中逐张找出最小的牌，依次排列。

## 3. 知识点

### 3.1 核心思想

```
选择排序过程:

初始: [64, 25, 12, 22, 11]

第1轮（找最小放到位置0）:
  最小值 = 11（索引4）
  交换 64 和 11 → [11, 25, 12, 22, 64]
  已排序: [11]

第2轮（找最小放到位置1）:
  剩余: [25, 12, 22, 64]
  最小值 = 12（索引2）
  交换 25 和 12 → [11, 12, 25, 22, 64]
  已排序: [11, 12]

第3轮（找最小放到位置2）:
  剩余: [25, 22, 64]
  最小值 = 22（索引3）
  交换 25 和 22 → [11, 12, 22, 25, 64]
  已排序: [11, 12, 22]

第4轮（找最小放到位置3）:
  剩余: [25, 64]
  最小值 = 25（索引3）
  无需交换 → [11, 12, 22, 25, 64]
  已排序: [11, 12, 22, 25]

最终结果: [11, 12, 22, 25, 64]
```

### 3.2 不稳定的原因

```
选择排序的不稳定性:

初始: [5a, 8, 5b, 2, 9]  (5a 和 5b 值相同，但 5a 在原序列中在前)

第1轮: 最小值 = 2（索引3）
       交换 5a 和 2 → [2, 8, 5b, 5a, 9]
       
       5a 从索引0 移到了索引3
       5b 在索引2
       
后续排序中，5b 和 5a 的相对顺序可能改变！
```

## 4. 数据结构

选择排序直接在输入数组上操作：

- 输入：`int[]`
- 原地排序：是
- 只需一个临时变量用于交换

```java
public class SelectionSort {
    public static void sort(int[] arr) {
        if (arr == null || arr.length < 2) return;
        
        int n = arr.length;
        for (int i = 0; i < n - 1; i++) {
            int minIndex = i;
            
            // 在未排序部分找最小值
            for (int j = i + 1; j < n; j++) {
                if (arr[j] < arr[minIndex]) {
                    minIndex = j;
                }
            }
            
            // 将最小值放到已排序部分的末尾
            if (minIndex != i) {
                int temp = arr[i];
                arr[i] = arr[minIndex];
                arr[minIndex] = temp;
            }
        }
    }
}
```

## 5. 最佳实践

1. **实际工程中很少使用，仅教学用途**。  
2. **比较次数固定为 n(n-1)/2，不受数据分布影响**。  
3. **交换次数最多 n-1 次，适合写入代价高的场景**。  
4. **不稳定排序，不适合需要稳定性的场景**。  
5. **通常被插入排序或快速排序替代**。
