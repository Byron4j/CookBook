# 贪心算法

## 1. 概念

贪心算法（Greedy Algorithm）是一种在每一步选择中都采取在当前状态下最好或最优的选择，从而希望导致结果是全局最好或最优的算法。

```
贪心算法核心思想:

问题: 找零问题，用最少的硬币凑出金额

硬币面额: [1, 2, 5, 10]
目标金额: 18

贪心策略: 每次选择不超过剩余金额的最大面额

Step 1: 剩余 18，选 10 → 剩余 8，硬币 [10]
Step 2: 剩余 8，选 5 → 剩余 3，硬币 [10, 5]
Step 3: 剩余 3，选 2 → 剩余 1，硬币 [10, 5, 2]
Step 4: 剩余 1，选 1 → 剩余 0，硬币 [10, 5, 2, 1]

结果: 4 枚硬币
最优解: 10 + 5 + 2 + 1 = 18 ✓

注意: 贪心不总是最优！
反例: 硬币 [1, 3, 4]，目标 6
贪心: 4 + 1 + 1 = 3 枚
最优: 3 + 3 = 2 枚
```

## 2. 来源

贪心算法思想源于日常生活中"短视"但有效的决策策略。  
虽然不一定全局最优，但在很多问题中能得到最优或近似最优解，且实现简单高效。

## 3. 知识点

### 3.1 贪心算法的适用条件

```
贪心选择性质（Greedy Choice Property）:

局部最优选择能导致全局最优解。

示例: 活动选择问题
有 n 个活动，每个活动有开始和结束时间，
选择尽可能多的互不重叠的活动。

贪心策略: 每次选择结束时间最早的活动

证明:
假设最优解第一个活动不是结束最早的，
用结束最早的活动替换它，不会减少活动数量。
因此贪心选择是正确的。

最优子结构（Optimal Substructure）:

问题的最优解包含子问题的最优解。

示例: 最小生成树
如果 e 是连接两个连通分量的最小权边，
那么 e 一定在某个最小生成树中。
```

### 3.2 经典例题：活动选择

```
问题: 给定 n 个活动的开始和结束时间，选择最多互不重叠的活动。

活动列表:
活动:  1  2  3  4  5  6
开始:  1  3  0  5  3  5
结束:  2  4  6  7  8  9

贪心策略: 按结束时间排序，每次选择结束最早且不冲突的活动

Step 1: 按结束时间排序
活动 1 (1,2) ← 结束最早
活动 2 (3,4)
活动 4 (5,7)
活动 3 (0,6)
活动 5 (3,8)
活动 6 (5,9)

Step 2: 选择活动 1 (结束时间 2)
        排除所有开始时间 < 2 的活动（无）

Step 3: 选择活动 2 (结束时间 4)
        排除所有开始时间 < 4 的活动（活动 3, 5）

Step 4: 选择活动 4 (结束时间 7)
        排除所有开始时间 < 7 的活动（活动 6）

结果: 活动 1, 2, 4，共 3 个
验证: 1-2, 3-4, 5-7 确实互不重叠 ✓
```

### 3.3 经典例题：霍夫曼编码

```
问题: 给定字符及其出现频率，构造最优前缀编码。

字符频率:
A: 45%, B: 13%, C: 12%, D: 16%, E: 9%, F: 5%

贪心策略: 每次合并频率最小的两个节点

构建过程:

Step 1: 将所有节点放入优先队列
        [F:5, E:9, C:12, B:13, D:16, A:45]

Step 2: 取出 F(5) 和 E(9)，合并为 FE(14)
        [C:12, B:13, D:16, FE:14, A:45]

Step 3: 取出 C(12) 和 B(13)，合并为 CB(25)
        [D:16, FE:14, CB:25, A:45]

Step 4: 取出 D(16) 和 FE(14)，合并为 DFE(30)
        [CB:25, DFE:30, A:45]

Step 5: 取出 CB(25) 和 DFE(30)，合并为 CBDFE(55)
        [A:45, CBDFE:55]

Step 6: 取出 A(45) 和 CBDFE(55)，合并为根(100)
        [根:100]

构建的树:

              100
             /   \
           A:45  55
                /  \
              25   30
             / \   / \
          C:12 B:13 D:16 14
                          / \
                        F:5 E:9

编码（左0右1）:
A: 0
B: 100
C: 101
D: 110
E: 1111
F: 1110

带权路径长度:
45×1 + 13×3 + 12×3 + 16×3 + 9×4 + 5×4 = 224
这是最优的前缀编码！
```

## 4. 数据结构

```java
public class GreedyAlgorithm {
    
    // 活动选择问题
    static class Activity {
        int start, end;
        Activity(int s, int e) { start = s; end = e; }
    }
    
    public static List<Activity> activitySelection(List<Activity> activities) {
        // 按结束时间排序
        activities.sort(Comparator.comparingInt(a -> a.end));
        
        List<Activity> result = new ArrayList<>();
        if (activities.isEmpty()) return result;
        
        // 选择第一个活动
        result.add(activities.get(0));
        int lastEnd = activities.get(0).end;
        
        // 选择后续不冲突的活动
        for (int i = 1; i < activities.size(); i++) {
            if (activities.get(i).start >= lastEnd) {
                result.add(activities.get(i));
                lastEnd = activities.get(i).end;
            }
        }
        
        return result;
    }
    
    // 找零问题（贪心版）
    public static List<Integer> makeChange(int[] coins, int amount) {
        // 降序排序
        Arrays.sort(coins);
        List<Integer> result = new ArrayList<>();
        
        for (int i = coins.length - 1; i >= 0; i--) {
            while (amount >= coins[i]) {
                amount -= coins[i];
                result.add(coins[i]);
            }
        }
        
        return amount == 0 ? result : null;  // 可能无法凑出
    }
    
    // 霍夫曼编码
    static class HuffmanNode {
        char ch;
        int freq;
        HuffmanNode left, right;
        
        HuffmanNode(char c, int f) {
            ch = c; freq = f;
        }
        
        HuffmanNode(int f, HuffmanNode l, HuffmanNode r) {
            freq = f; left = l; right = r;
        }
    }
    
    public static HuffmanNode buildHuffmanTree(Map<Character, Integer> freqMap) {
        PriorityQueue<HuffmanNode> pq = new PriorityQueue<>(
            Comparator.comparingInt(n -> n.freq)
        );
        
        // 初始化优先队列
        for (Map.Entry<Character, Integer> entry : freqMap.entrySet()) {
            pq.offer(new HuffmanNode(entry.getKey(), entry.getValue()));
        }
        
        // 合并节点
        while (pq.size() > 1) {
            HuffmanNode left = pq.poll();
            HuffmanNode right = pq.poll();
            HuffmanNode parent = new HuffmanNode(
                left.freq + right.freq, left, right
            );
            pq.offer(parent);
        }
        
        return pq.poll();
    }
}
```

## 5. 最佳实践

1. **先证明贪心选择的正确性**，再实现算法。  
2. **贪心不总是最优**，必要时使用动态规划验证。  
3. **优先队列常用于贪心算法的实现**。  
4. **排序是很多贪心问题的前置步骤**。  
5. **贪心算法的证明方法**：交换论证、归纳法。

### 贪心 vs 动态规划

```
对比维度:

                    贪心算法              动态规划
决策方式:           局部最优              全局最优
时间复杂度:         通常 O(n) 或 O(nlogn)  O(n²) 或更高
空间复杂度:         通常 O(1) 或 O(n)      通常 O(n) 或 O(n²)
适用场景:           具有贪心选择性质      具有重叠子问题
正确性:            需要证明               只要方程正确即正确

选择建议:
- 如果能证明贪心选择正确 → 贪心（更快更简单）
- 如果不能确定 → 动态规划（更通用）
- 可以先尝试贪心，再用 DP 验证
```

### 实际应用

```
实际应用场景:

1. 调度问题
   - 任务调度
   - CPU 调度算法

2. 数据压缩
   - 霍夫曼编码
   - 文件压缩

3. 图算法
   - 最小生成树（Prim、Kruskal）
   - Dijkstra 最短路径

4. 资源分配
   - 分数背包问题
   - 带宽分配

5. 近似算法
   - 当问题 NP-hard 时，贪心可得到近似解
   - 集合覆盖、旅行商问题等
```
