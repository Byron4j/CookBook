# 冒泡排序算法

## 1. 概念

冒泡排序（Bubble Sort）是一种简单的交换排序算法，通过重复遍历数组，比较相邻元素并交换位置来排序。

- 平均时间复杂度：`O(n²)`
- 最坏时间复杂度：`O(n²)`
- 最好时间复杂度：`O(n)`（已排序时）
- 空间复杂度：`O(1)`（原地排序）
- 稳定性：**稳定排序**

## 2. 来源

冒泡排序是最基础的排序算法之一，名字来源于大元素像气泡一样逐渐"浮"到数组末尾。

## 3. 知识点

### 3.1 核心思想

```
冒泡排序过程:

初始: [5, 3, 8, 4, 2]

第1轮（最大的元素"浮"到最后）:
  比较 5,3 → 交换 → [3, 5, 8, 4, 2]
  比较 5,8 → 不交换 → [3, 5, 8, 4, 2]
  比较 8,4 → 交换 → [3, 5, 4, 8, 2]
  比较 8,2 → 交换 → [3, 5, 4, 2, 8]
  结果: 8 已在正确位置

第2轮（次大的元素"浮"到倒数第二）:
  比较 3,5 → 不交换 → [3, 5, 4, 2, 8]
  比较 5,4 → 交换 → [3, 4, 5, 2, 8]
  比较 5,2 → 交换 → [3, 4, 2, 5, 8]
  结果: 5 已在正确位置

第3轮:
  比较 3,4 → 不交换 → [3, 4, 2, 5, 8]
  比较 4,2 → 交换 → [3, 2, 4, 5, 8]
  结果: 4 已在正确位置

第4轮:
  比较 3,2 → 交换 → [2, 3, 4, 5, 8]
  结果: 完成

最终结果: [2, 3, 4, 5, 8]
```

### 3.2 优化

```
优化 1: 设置标志位

如果一轮中没有发生交换，说明数组已有序，可以提前结束

优化后的流程:

初始: [1, 2, 3, 5, 4]

第1轮:
  比较 1,2 → 不交换
  比较 2,3 → 不交换
  比较 3,5 → 不交换
  比较 5,4 → 交换 → [1, 2, 3, 4, 5]
  发生交换，继续

第2轮:
  比较 1,2 → 不交换
  比较 2,3 → 不交换
  比较 3,4 → 不交换
  比较 4,5 → 不交换
  未发生交换，提前结束！

时间复杂度（最好情况）: O(n)


优化 2: 记录最后交换位置

每轮记录最后一次交换的位置，下一轮只需比较到该位置

初始: [3, 2, 1, 5, 4]

第1轮:
  ...
  最后交换位置 = 3（5 和 4 交换）
  下一轮只需比较到索引 3
```

## 4. 数据结构

冒泡排序直接在输入数组上操作：

- 输入：`int[]`
- 原地排序：是
- 只需一个临时变量用于交换

```java
public class BubbleSort {
    public static void sort(int[] arr) {
        if (arr == null || arr.length < 2) return;
        
        int n = arr.length;
        for (int i = 0; i < n - 1; i++) {
            boolean swapped = false;
            
            for (int j = 0; j < n - 1 - i; j++) {
                if (arr[j] > arr[j + 1]) {
                    // 交换
                    int temp = arr[j];
                    arr[j] = arr[j + 1];
                    arr[j + 1] = temp;
                    swapped = true;
                }
            }
            
            // 如果没有交换，提前结束
            if (!swapped) break;
        }
    }
}
```

## 5. 最佳实践

1. **教学用算法，实际工程中很少使用**。  
2. **数据量极小（n < 10）且已接近有序时可考虑**。  
3. **稳定排序需求且数据量小时可用**。  
4. **通常被插入排序替代**。  
5. **面试中常考基础实现和优化**。
