# 堆排序算法

## 1. 概念

堆排序（Heap Sort）是利用堆这种数据结构设计的排序算法。

- 平均时间复杂度：`O(n log n)`
- 最坏时间复杂度：`O(n log n)`
- 空间复杂度：`O(1)`（原地排序！）
- 稳定性：**不稳定排序**

## 2. 来源

堆排序由 J. W. J. Williams 于 1964 年提出。  
它解决了快速排序最坏情况和归并排序空间复杂度的问题。

## 3. 知识点

### 3.1 堆的定义

```
大顶堆（Max Heap）:

每个节点的值 ≥ 其子节点的值

            [90]           ← 根节点最大
           /    \
        [80]    [70]
        /  \    /  \
     [60] [50][40] [30]
     /
   [20]

数组表示: [90, 80, 70, 60, 50, 40, 30, 20]

节点关系:
- 节点 i 的左子节点: 2i + 1
- 节点 i 的右子节点: 2i + 2
- 节点 i 的父节点: (i-1) / 2

验证:
索引 0 (90): 子节点 1(80), 2(70) → 90 ≥ 80, 90 ≥ 70 ✓
索引 1 (80): 子节点 3(60), 4(50) → 80 ≥ 60, 80 ≥ 50 ✓
索引 3 (60): 子节点 7(20) → 60 ≥ 20 ✓
```

### 3.2 堆化（Heapify）

```
从最后一个非叶子节点开始向下调整:

原始数组: [3, 5, 1, 10, 2]

Step 1: 找到最后一个非叶子节点
        n = 5, 最后一个非叶子节点索引 = (5/2) - 1 = 1 (值为5)

Step 2: 对索引 1 进行堆化
        节点 5, 左子节点 10, 右子节点 2
        5 < 10 → 交换 5 和 10
        数组: [3, 10, 1, 5, 2]
        
        检查是否需要继续下沉:
        节点 5(现在索引3), 无子节点 → 停止

Step 3: 对索引 0 进行堆化
        节点 3, 左子节点 10, 右子节点 1
        3 < 10 → 交换 3 和 10
        数组: [10, 3, 1, 5, 2]
        
        节点 3(现在索引1), 左子节点 5, 右子节点 2
        3 < 5 → 交换 3 和 5
        数组: [10, 5, 1, 3, 2]
        
        节点 3(现在索引3), 无子节点 → 停止

结果大顶堆: [10, 5, 1, 3, 2]
```

### 3.3 堆排序流程

```
堆排序过程:

初始: [4, 10, 3, 5, 1]

Step 1: 构建大顶堆
        [10, 5, 3, 4, 1]
        
Step 2: 排序过程

第1轮:
  交换堆顶和末尾: [1, 5, 3, 4, 10]
  对前 4 个元素堆化: [5, 4, 3, 1, 10]
  结果: [5, 4, 3, 1] + [10]

第2轮:
  交换堆顶和末尾: [1, 4, 3, 5, 10]
  对前 3 个元素堆化: [4, 1, 3, 5, 10]
  结果: [4, 1, 3] + [5, 10]

第3轮:
  交换堆顶和末尾: [3, 1, 4, 5, 10]
  对前 2 个元素堆化: [3, 1, 4, 5, 10]
  结果: [3, 1] + [4, 5, 10]

第4轮:
  交换堆顶和末尾: [1, 3, 4, 5, 10]
  对前 1 个元素堆化: [1, 3, 4, 5, 10]
  结果: [1] + [3, 4, 5, 10]

最终结果: [1, 3, 4, 5, 10]（升序）
```

### 3.4 下沉操作（Sift Down）

```
siftDown 流程:

当前节点索引 i，堆大小 n:

1. 找到左子节点: left = 2i + 1
2. 找到右子节点: right = 2i + 2
3. largest = i
4. 如果 left < n 且 arr[left] > arr[largest]:
      largest = left
5. 如果 right < n 且 arr[right] > arr[largest]:
      largest = right
6. 如果 largest ≠ i:
      交换 arr[i] 和 arr[largest]
      siftDown(largest, n)  // 继续下沉
```

## 4. 数据结构

堆排序直接在输入数组上操作：

- 输入：`int[]`
- 原地排序：是
- 无需额外数组

```java
public class HeapSort {
    public static void sort(int[] arr) {
        if (arr == null || arr.length < 2) return;
        
        int n = arr.length;
        
        // 构建大顶堆（从最后一个非叶子节点开始）
        for (int i = n / 2 - 1; i >= 0; i--) {
            siftDown(arr, i, n);
        }
        
        // 排序
        for (int i = n - 1; i > 0; i--) {
            // 交换堆顶和当前末尾
            swap(arr, 0, i);
            // 对前 i 个元素堆化
            siftDown(arr, 0, i);
        }
    }
    
    private static void siftDown(int[] arr, int i, int n) {
        int largest = i;
        int left = 2 * i + 1;
        int right = 2 * i + 2;
        
        if (left < n && arr[left] > arr[largest])
            largest = left;
        if (right < n && arr[right] > arr[largest])
            largest = right;
        
        if (largest != i) {
            swap(arr, i, largest);
            siftDown(arr, largest, n);
        }
    }
    
    private static void swap(int[] arr, int i, int j) {
        int temp = arr[i];
        arr[i] = arr[j];
        arr[j] = temp;
    }
}
```

## 5. 最佳实践

1. **内存受限时优先堆排序**（原地排序）。  
2. **优先队列（PriorityQueue）底层就是堆**。  
3. **Top K 问题可以用堆优化到 O(n log k)**。  
4. **合并 K 个有序数组用堆**。  
5. **不稳定排序，不适合需要稳定性的场景**。

### 堆的应用

```
优先队列实现:

操作          时间复杂度
插入元素       O(log n)
取出最大元素   O(log n)
查看最大元素   O(1)

应用场景:
- 任务调度（按优先级）
- 合并 K 个有序数组
- Top K 问题
- Dijkstra 最短路径

Top K 问题:

维护大小为 K 的小顶堆:
- 遍历数组
- 如果堆大小 < K，直接加入
- 如果当前元素 > 堆顶，替换堆顶
- 最终堆中就是最大的 K 个元素

时间复杂度: O(n log K)
空间复杂度: O(K)
```
