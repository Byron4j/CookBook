# 快速排序算法

## 1. 概念

快速排序（Quick Sort）是一种分治排序算法。  
它通过"选择基准值 + 分区"不断把问题拆小，最终完成整体有序。

```
核心思想 - 分治:

原始数组: [6, 3, 8, 1, 9, 2, 7, 5, 4]

Step 1: 选择基准值 pivot = 6

Step 2: 分区（Partition）
        小于 6 放左边      大于 6 放右边
        ↓                ↓
   [3, 1, 2, 5, 4]   [6]   [8, 9, 7]

Step 3: 递归排序左右子数组
   [3, 1, 2, 5, 4] → pivot=3 → [1, 2] [3] [5, 4]
   [8, 9, 7] → pivot=8 → [7] [8] [9]

最终结果: [1, 2, 3, 4, 5, 6, 7, 8, 9]
```

平均时间复杂度：`O(n log n)`  
最坏时间复杂度：`O(n²)`  
空间复杂度（递归栈）：平均 `O(log n)`

## 2. 来源

快速排序由 Tony Hoare 在 1960 年提出。  
它在工程中长期流行，原因是常数项小、缓存友好、实现灵活，通常在通用场景下性能表现优异。

## 3. 知识点

### 3.1 分区过程详解

```
分区算法（Lomuto 方案）:

数组: [6, 3, 8, 1, 9, 2, 7, 5, 4]
pivot = 6（取最后一个元素）

初始化: i = -1（小于区的右边界）

遍历 j = 0 到 n-2:

j=0: arr[0]=6 ≥ pivot=6，不交换
      i=-1, j=0
      [6, 3, 8, 1, 9, 2, 7, 5, 4]

j=1: arr[1]=3 < pivot=6，i++，交换 arr[i] 和 arr[j]
      i=0, j=1
      交换 arr[0] 和 arr[1]: [3, 6, 8, 1, 9, 2, 7, 5, 4]

j=2: arr[2]=8 ≥ pivot，不交换
      i=0, j=2

j=3: arr[3]=1 < pivot，i++，交换
      i=1, j=3
      交换 arr[1] 和 arr[3]: [3, 1, 8, 6, 9, 2, 7, 5, 4]

j=4: arr[4]=9 ≥ pivot，不交换

j=5: arr[5]=2 < pivot，i++，交换
      i=2, j=5
      交换 arr[2] 和 arr[5]: [3, 1, 2, 6, 9, 8, 7, 5, 4]

j=6: arr[6]=7 ≥ pivot，不交换

j=7: arr[7]=5 < pivot，i++，交换
      i=3, j=7
      交换 arr[3] 和 arr[7]: [3, 1, 2, 5, 9, 8, 7, 6, 4]

最后: 交换 arr[i+1] 和 arr[n-1] (pivot)
      i=3
      交换 arr[4] 和 arr[8]: [3, 1, 2, 5, 4, 8, 7, 6, 9]
      
结果: 左边 [3,1,2,5,4] < 6 < 右边 [8,7,6,9]
注意: pivot 现在在索引 4 的位置
```

### 3.2 基准选择策略

```
不同 pivot 选择策略:

1. 固定取首元素:
   [1, 2, 3, 4, 5, 6, 7, 8, 9]
   pivot=1，分区极度不平衡！
   时间复杂度退化到 O(n²)

2. 固定取尾元素:
   [9, 8, 7, 6, 5, 4, 3, 2, 1]
   pivot=1，同样极度不平衡！

3. 随机选择:
   避免最坏情况，期望 O(n log n)
   实现: int randomIndex = low + random.nextInt(high - low + 1)

4. 三数取中（Median-of-Three）:
   取首、中、尾三个元素的中位数
   兼顾性能和实现复杂度
   例如: [6, 3, 8] → 中位数是 6
```

### 3.3 复杂度分析

```
时间复杂度分析:

最优情况（每次都平衡划分）:
T(n) = 2T(n/2) + O(n) = O(n log n)

      [n]
     /   \
  [n/2] [n/2]
  /  \   /  \
递归树高度: log₂n
每层工作量: O(n)
总工作量: O(n log n)

最坏情况（极度不平衡）:
T(n) = T(n-1) + O(n) = O(n²)

      [n]
       |
     [n-1]
       |
     [n-2]
       |
      ...
      
递归树高度: n
每层工作量: O(n)
总工作量: O(n²)

平均情况: O(n log n)
```

## 4. 数据结构

快速排序不依赖复杂结构，直接作用于数组：

- 输入：`int[]`
- 原地交换，减少额外内存开销

代码位置：`src/main/java/org/byron4j/cookbook/algrithms/QuickSort.java`

当前实现新增了两个入口：

- `sort(int[] array)`：对原数组排序
- `sortedCopy(int[] array)`：返回排序后的副本

```java
public class QuickSort {
    
    public static void sort(int[] arr) {
        quickSort(arr, 0, arr.length - 1);
    }
    
    private static void quickSort(int[] arr, int low, int high) {
        if (low < high) {
            int pivotIndex = partition(arr, low, high);
            quickSort(arr, low, pivotIndex - 1);
            quickSort(arr, pivotIndex + 1, high);
        }
    }
    
    private static int partition(int[] arr, int low, int high) {
        int pivot = arr[high];  // 选最后一个为 pivot
        int i = low - 1;  // 小于 pivot 的元素的边界
        
        for (int j = low; j < high; j++) {
            if (arr[j] <= pivot) {
                i++;
                swap(arr, i, j);
            }
        }
        
        swap(arr, i + 1, high);  // pivot 放到正确位置
        return i + 1;
    }
    
    private static void swap(int[] arr, int i, int j) {
        int temp = arr[i];
        arr[i] = arr[j];
        arr[j] = temp;
    }
}
```

## 5. 最佳实践

1. **业务代码优先封装统一入口**，避免直接暴露递归细节。  
2. **对空数组、单元素数组做快速返回**。  
3. **大规模数据建议引入随机基准**，降低退化概率。  
4. **需要稳定排序时不要用快排**，改用归并排序。  
5. **始终用自动化测试覆盖边界值和重复值场景**。

### 排序算法对比

```
各种排序算法对比:

算法        平均时间    最坏时间    空间      稳定性    适用场景
─────────────────────────────────────────────────────────────
快速排序    O(nlogn)   O(n²)      O(logn)   不稳定   通用，最快
归并排序    O(nlogn)   O(nlogn)   O(n)      稳定     链表排序
堆排序      O(nlogn)   O(nlogn)   O(1)      不稳定   内存受限
插入排序    O(n²)      O(n²)      O(1)      稳定     小数据/近乎有序
选择排序    O(n²)      O(n²)      O(1)      不稳定   理论教学
冒泡排序    O(n²)      O(n²)      O(1)      稳定     教学用
计数排序    O(n+k)     O(n+k)     O(k)      稳定     整数范围小

Java 实际使用:
- Arrays.sort(int[]): Dual-Pivot QuickSort
- Arrays.sort(Object[]): TimSort（归并排序优化版）
- Collections.sort(): 基于 TimSort
```
