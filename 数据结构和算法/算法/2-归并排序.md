# 归并排序算法

## 1. 概念

归并排序（Merge Sort）是一种分治排序算法，核心思想是"先分后合"。

- 平均时间复杂度：`O(n log n)`
- 最坏时间复杂度：`O(n log n)`（稳定！）
- 空间复杂度：`O(n)`（需要额外数组）
- 稳定性：**稳定排序**

## 2. 来源

归并排序由 John von Neumann 于 1945 年提出。  
它是最早的排序算法之一，特点是性能稳定，不受数据分布影响。

## 3. 知识点

### 3.1 核心思想

```
分治策略:

        [8,4,5,7,1,3,6,2]
              /   \
       [8,4,5,7]   [1,3,6,2]
        /    \      /    \
     [8,4]  [5,7] [1,3]  [6,2]
      / \    / \   / \    / \
    [8][4][5][7][1][3][6][2]
      \  /  \  /  \  /  \  /
     [4,8]  [5,7] [1,3]  [2,6]
        \     /      \     /
       [4,5,7,8]    [1,2,3,6]
            \         /
         [1,2,3,4,5,6,7,8]

分: 递归将数组二分，直到只有一个元素
治: 合并两个有序数组
```

### 3.2 合并过程详解

```
合并两个有序数组:

左数组: [4, 8]      右数组: [5, 7]

Step 1: 比较 4 和 5，取 4
结果: [4]
左: [8]  右: [5,7]

Step 2: 比较 8 和 5，取 5
结果: [4, 5]
左: [8]  右: [7]

Step 3: 比较 8 和 7，取 7
结果: [4, 5, 7]
左: [8]  右: []

Step 4: 右数组为空，将左数组剩余元素加入
结果: [4, 5, 7, 8]

合并完成！
```

## 4. 数据结构

归并排序需要额外数组来合并：

- 输入：`int[]`
- 辅助数组：`int[] temp`（大小为 n）
- 原地交换：否，需要额外空间

```java
public class MergeSort {
    public static void sort(int[] arr) {
        if (arr == null || arr.length < 2) return;
        int[] temp = new int[arr.length];
        mergeSort(arr, 0, arr.length - 1, temp);
    }
    
    private static void mergeSort(int[] arr, int left, int right, int[] temp) {
        if (left < right) {
            int mid = left + (right - left) / 2;
            mergeSort(arr, left, mid, temp);      // 排序左半
            mergeSort(arr, mid + 1, right, temp); // 排序右半
            merge(arr, left, mid, right, temp);   // 合并
        }
    }
    
    private static void merge(int[] arr, int left, int mid, int right, int[] temp) {
        int i = left;      // 左数组指针
        int j = mid + 1;   // 右数组指针
        int k = 0;         // temp 数组指针
        
        // 比较并放入 temp
        while (i <= mid && j <= right) {
            if (arr[i] <= arr[j]) {
                temp[k++] = arr[i++];
            } else {
                temp[k++] = arr[j++];
            }
        }
        
        // 左数组剩余
        while (i <= mid) temp[k++] = arr[i++];
        // 右数组剩余
        while (j <= right) temp[k++] = arr[j++];
        
        // 复制回原数组
        for (int t = 0; t < k; t++) {
            arr[left + t] = temp[t];
        }
    }
}
```

## 5. 最佳实践

1. **需要稳定排序时优先归并排序**。  
2. **链表排序可用归并，无需额外空间**。  
3. **小规模数据（n < 16）可切换插入排序优化**。  
4. **多线程环境下可用并行归并排序**。  
5. **外部排序（大文件）的核心算法**。  

### 适用场景对比

```
排序算法选择指南:

数据规模小 (n < 100): 插入排序 ✓
数据基本有序: 插入排序 ✓
需要稳定排序: 归并排序 ✓
数据规模中等 (n < 1M): 快速排序 ✓
数据规模很大 (n > 1M): 归并排序 / 堆排序 ✓
内存受限: 堆排序 ✓
```
