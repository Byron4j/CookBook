# 插入排序算法

## 1. 概念

插入排序（Insertion Sort）是一种简单直观的排序算法，通过构建有序序列，对于未排序数据，在已排序序列中从后向前扫描，找到相应位置并插入。

- 平均时间复杂度：`O(n²)`
- 最坏时间复杂度：`O(n²)`
- 最好时间复杂度：`O(n)`（已排序时）
- 空间复杂度：`O(1)`（原地排序）
- 稳定性：**稳定排序**

## 2. 来源

插入排序类似于整理扑克牌：将每张新牌插入到已排序牌组的正确位置。

## 3. 知识点

### 3.1 核心思想

```
插入排序过程:

初始: [5, 2, 4, 6, 1, 3]

第1轮（处理元素2）:
  已排序: [5]
  将 2 插入到 [5] 中:
    2 < 5 → 5 后移 → [5, 5, 4, 6, 1, 3]
    到达开头，放入 2 → [2, 5, 4, 6, 1, 3]

第2轮（处理元素4）:
  已排序: [2, 5]
  将 4 插入到 [2, 5] 中:
    4 < 5 → 5 后移 → [2, 5, 5, 6, 1, 3]
    4 > 2 → 放入 4 → [2, 4, 5, 6, 1, 3]

第3轮（处理元素6）:
  已排序: [2, 4, 5]
  将 6 插入到 [2, 4, 5] 中:
    6 > 5 → 直接放到末尾 → [2, 4, 5, 6, 1, 3]

第4轮（处理元素1）:
  已排序: [2, 4, 5, 6]
  将 1 插入:
    1 < 6 → 6 后移
    1 < 5 → 5 后移
    1 < 4 → 4 后移
    1 < 2 → 2 后移
    到达开头，放入 1 → [1, 2, 4, 5, 6, 3]

第5轮（处理元素3）:
  已排序: [1, 2, 4, 5, 6]
  将 3 插入:
    3 < 6 → 6 后移
    3 < 5 → 5 后移
    3 < 4 → 4 后移
    3 > 2 → 放入 3 → [1, 2, 3, 4, 5, 6]

最终结果: [1, 2, 3, 4, 5, 6]
```

### 3.2 与冒泡排序的对比

```
相似点:
- 时间复杂度相同: O(n²)
- 空间复杂度相同: O(1)
- 最好情况都是 O(n)

不同点:
- 插入排序比较次数更少（平均）
- 插入排序赋值操作更少
- 插入排序在部分有序时性能更好

实际测试（n=10000，随机数据）:
冒泡排序: ~500ms
插入排序: ~120ms
快速排序: ~2ms
```

### 3.3 二分插入排序

```
优化：用二分查找找到插入位置

原始插入排序:
  从后向前线性查找插入位置: O(n)

二分插入排序:
  用二分查找找到插入位置: O(log n)
  但元素后移仍需要 O(n)
  总时间复杂度: O(n²)（常数优化）

适用场景:
- 比较操作代价高时（如字符串比较）
- 数据移动代价低时（如链表）

```

## 4. 数据结构

插入排序直接在输入数组上操作：

- 输入：`int[]`
- 原地排序：是
- 只需一个临时变量保存当前元素

```java
public class InsertionSort {
    public static void sort(int[] arr) {
        if (arr == null || arr.length < 2) return;
        
        for (int i = 1; i < arr.length; i++) {
            int key = arr[i];  // 当前要插入的元素
            int j = i - 1;
            
            // 将大于 key 的元素后移
            while (j >= 0 && arr[j] > key) {
                arr[j + 1] = arr[j];
                j--;
            }
            
            // 插入到正确位置
            arr[j + 1] = key;
        }
    }
}
```

## 5. 最佳实践

1. **小规模数据（n < 50）时性能优异**。  
2. **数据基本有序时性能接近 O(n)**。  
3. **常被用作快速排序/归并排序的 fallback**。  
4. **在线算法**：可以边接收数据边排序。  
5. **链表排序的自然选择**。  

### 适用场景

```
排序算法选择决策树:

n < 50?
  ├── 是 → 插入排序 ✓
  │         - 简单高效
  │         - 常数小
  │
  └── 否 → 基本有序?
             ├── 是 → 插入排序 ✓
             │         - 接近 O(n)
             │
             └── 否 → 内存充足?
                        ├── 是 → 归并排序 ✓
                        │         - 稳定
                        │         - 性能好
                        │
                        └── 否 → 快速排序 ✓
                                  - 平均最快
                                  - 原地排序
```
