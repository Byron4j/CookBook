# 广度优先搜索 (BFS)

## 1. 概念

广度优先搜索（Breadth-First Search，BFS）是一种用于遍历或搜索树/图的算法。  
它从根节点（或任意起始节点）开始，逐层遍历节点，先访问所有相邻节点，再访问相邻节点的相邻节点。

```
BFS 遍历过程示例:

图结构:
        1
       / \
      2   3
     / \   \
    4   5   6

BFS 遍历顺序:
1 → 2 → 3 → 4 → 5 → 6

层级遍历可视化:

第 1 层: [1]
         │
         ├──→ 第 2 层: [2, 3]
                      │    │
                      ↓    ↓
                第 3 层: [4, 5, 6]

访问过程:

Step 1: 访问 1（第1层）
        队列: [1]
        
Step 2: 1 出队，邻居 2, 3 入队（第2层）
        访问: [1]
        队列: [2, 3]
        
Step 3: 2 出队，邻居 4, 5 入队（第3层）
        访问: [1, 2]
        队列: [3, 4, 5]
        
Step 4: 3 出队，邻居 6 入队（第3层）
        访问: [1, 2, 3]
        队列: [4, 5, 6]
        
Step 5: 4 出队，无新邻居
        访问: [1, 2, 3, 4]
        队列: [5, 6]
        
Step 6: 5 出队，无新邻居
        访问: [1, 2, 3, 4, 5]
        队列: [6]
        
Step 7: 6 出队，无新邻居
        访问: [1, 2, 3, 4, 5, 6]
        队列: []
        
完成！
```

## 2. 来源

BFS 是图论中另一种基础遍历算法，与 DFS 互补。  
由于 BFS 按层级扩展，天然适合求解最短路径问题。

## 3. 知识点

### 3.1 算法流程

```
BFS 算法流程:

BFS(start):
    queue = [start]      ← 队列，FIFO
    visited = {start}    ← 已访问集合
    
    while queue 不为空:
        node = queue.dequeue()  ← 出队
        访问 node
        
        for node 的每个邻接节点 neighbor:
            if neighbor 未被访问:
                标记 neighbor 为已访问
                queue.enqueue(neighbor)  ← 入队

关键特性:
- 队列保证先进先出，实现层级遍历
- 先访问的节点距离起点更近
- 适合找最短路径（无权图）
```

### 3.2 时间复杂度

```
时间复杂度:

邻接矩阵表示: O(V²)
- 每个节点访问一次
- 每次访问需要检查所有 V 个节点

邻接表表示: O(V + E)
- 每个节点访问一次
- 每条边访问一次（无向图）或两次（有向图）

空间复杂度:
- 队列: O(V)（最坏情况存储一层所有节点）
- visited 数组/集合: O(V)
```

### 3.3 BFS vs DFS

```
对比维度:

                    BFS                  DFS
数据结构:           队列                 栈（递归或显式）
搜索顺序:           层级扩展              深度优先
最短路径:           能找到（无权图）       不一定
内存占用:           较大（存储整层节点）    较小（只存储路径）
适用场景:           最短路径、层级遍历     路径搜索、拓扑排序

选择建议:
- 找最短路径 → BFS
- 检测环 → DFS
- 空间受限 → DFS
- 需要层级信息 → BFS
```

### 3.4 最短路径应用

```
无权图最短路径:

图结构:
    A --- B
    |     |
    C --- D --- E

从 A 到 E 的最短路径:

BFS 过程:

第 0 层: [A]
第 1 层: [B, C]
第 2 层: [D]
第 3 层: [E]

路径: A → B → D → E  或  A → C → D → E
长度: 3

BFS 保证找到的路径是最短的（边数最少）


记录路径的方法:

每个节点记录父节点:
A: null
B: A
C: A
D: B (或 C)
E: D

从 E 回溯:
E → D → B → A
反转得到路径: A → B → D → E
```

## 4. 数据结构

```java
// BFS 实现
public class BFS {
    
    // 基本 BFS
    public static void bfs(Graph graph, int start) {
        boolean[] visited = new boolean[graph.V];
        Queue<Integer> queue = new LinkedList<>();
        
        visited[start] = true;
        queue.offer(start);
        
        while (!queue.isEmpty()) {
            int node = queue.poll();
            System.out.print(node + " ");
            
            for (int neighbor : graph.adj[node]) {
                if (!visited[neighbor]) {
                    visited[neighbor] = true;
                    queue.offer(neighbor);
                }
            }
        }
    }
    
    // BFS 求最短路径（记录父节点）
    public static List<Integer> shortestPath(Graph graph, int start, int end) {
        boolean[] visited = new boolean[graph.V];
        int[] parent = new int[graph.V];  // 记录父节点
        Arrays.fill(parent, -1);
        
        Queue<Integer> queue = new LinkedList<>();
        visited[start] = true;
        queue.offer(start);
        
        while (!queue.isEmpty()) {
            int node = queue.poll();
            
            if (node == end) {
                // 构建路径
                List<Integer> path = new ArrayList<>();
                for (int at = end; at != -1; at = parent[at]) {
                    path.add(at);
                }
                Collections.reverse(path);
                return path;
            }
            
            for (int neighbor : graph.adj[node]) {
                if (!visited[neighbor]) {
                    visited[neighbor] = true;
                    parent[neighbor] = node;
                    queue.offer(neighbor);
                }
            }
        }
        
        return null;  // 无路径
    }
    
    // 层级遍历（按层输出）
    public static List<List<Integer>> levelOrder(Graph graph, int start) {
        List<List<Integer>> result = new ArrayList<>();
        boolean[] visited = new boolean[graph.V];
        Queue<Integer> queue = new LinkedList<>();
        
        visited[start] = true;
        queue.offer(start);
        
        while (!queue.isEmpty()) {
            int levelSize = queue.size();
            List<Integer> level = new ArrayList<>();
            
            for (int i = 0; i < levelSize; i++) {
                int node = queue.poll();
                level.add(node);
                
                for (int neighbor : graph.adj[node]) {
                    if (!visited[neighbor]) {
                        visited[neighbor] = true;
                        queue.offer(neighbor);
                    }
                }
            }
            
            result.add(level);
        }
        
        return result;
    }
}
```

## 5. 最佳实践

1. **找无权图最短路径时优先使用 BFS**。  
2. **标记已访问节点避免重复处理和死循环**。  
3. **需要层级信息时使用按层遍历**。  
4. **大图上注意队列可能占用大量内存**。  
5. **双向 BFS 可优化搜索空间**（从起点和终点同时搜索）。

### 实际应用：社交网络

```
社交网络中的 BFS:

用户 A 的一度好友: B, C, D
用户 A 的二度好友: E, F, G, H

BFS 遍历:
第 1 层: [A]
第 2 层: [B, C, D]
第 3 层: [E, F, G, H]

应用:
- 查找最短关系链: A → B → E
- 推荐好友: 二度好友中未添加的
- 影响力分析: 层级越近影响力越大
```
