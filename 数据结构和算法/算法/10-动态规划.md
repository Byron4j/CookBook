# 动态规划

## 1. 概念

动态规划（Dynamic Programming，DP）是一种通过把原问题分解为相对简单的子问题的方式求解复杂问题的方法。  
核心思想：记住已解决的子问题的答案，避免重复计算。

```
动态规划核心思想:

问题: 计算 Fibonacci(5)

递归方式（存在大量重复计算）:
                    fib(5)
                   /      \
               fib(4)    fib(3)
               /    \     /    \
           fib(3) fib(2) fib(2) fib(1)
           /    \
       fib(2) fib(1)
       
fib(3) 被计算了 2 次！
fib(2) 被计算了 3 次！

动态规划方式（记忆化）:

fib(0) = 0  → 存入 memo[0]
fib(1) = 1  → 存入 memo[1]
fib(2) = fib(1) + fib(0) = 1  → 存入 memo[2]
fib(3) = fib(2) + fib(1) = 2  → 存入 memo[3]
fib(4) = fib(3) + fib(2) = 3  → 存入 memo[4]
fib(5) = fib(4) + fib(3) = 5  → 存入 memo[5]

每个子问题只计算一次！
```

## 2. 来源

动态规划由 Richard Bellman 于 1950 年代提出。  
名称中的 "Programming" 并非指编程，而是指"规划"或"决策"。

## 3. 知识点

### 3.1 动态规划的三大特征

```
1. 最优子结构（Optimal Substructure）
   
   问题的最优解包含子问题的最优解。
   
   示例: 最短路径
   如果 A → C 的最短路径经过 B，
   那么 A → B 和 B → C 也是最短路径。

2. 重叠子问题（Overlapping Subproblems）
   
   子问题之间存在重叠，会被重复计算。
   
   示例: Fibonacci
   fib(5) 需要 fib(4) 和 fib(3)
   fib(4) 需要 fib(3) 和 fib(2)
   fib(3) 被重复计算！

3. 无后效性
   
   一旦某个阶段的状态确定，就不受之后决策的影响。
   
   示例: 背包问题
   前 i 个物品的最优解只与前 i-1 个物品有关，
   与具体选择顺序无关。
```

### 3.2 DP 解题步骤

```
标准解题流程:

Step 1: 定义状态
        确定 dp 数组的含义
        示例: dp[i] 表示前 i 个元素的最优解
        
Step 2: 状态转移方程
        找出 dp[i] 与 dp[i-1], dp[i-2], ... 的关系
        示例: dp[i] = max(dp[i-1], dp[i-2] + nums[i])
        
Step 3: 初始化
        确定初始状态的值
        示例: dp[0] = 0, dp[1] = nums[1]
        
Step 4: 遍历顺序
        确定计算的先后顺序
        示例: 从小到大，或从大到小
        
Step 5: 返回结果
        根据题意返回 dp 数组的某个值
        示例: return dp[n]
```

### 3.3 经典例题：爬楼梯

```
问题: 有 n 级台阶，每次可以爬 1 级或 2 级，求有多少种爬法。

状态定义:
dp[i] = 爬到第 i 级台阶的方法数

状态转移:
爬到第 i 级，可能是从 i-1 级爬 1 步，或从 i-2 级爬 2 步
dp[i] = dp[i-1] + dp[i-2]

初始化:
dp[0] = 1（地面，1种方法）
dp[1] = 1（只有1级，1种方法）

计算过程:
dp[2] = dp[1] + dp[0] = 1 + 1 = 2
        (1+1, 2)
        
dp[3] = dp[2] + dp[1] = 2 + 1 = 3
        (1+1+1, 1+2, 2+1)
        
dp[4] = dp[3] + dp[2] = 3 + 2 = 5
        (1+1+1+1, 1+1+2, 1+2+1, 2+1+1, 2+2)

结果: dp[n]
```

### 3.4 经典例题：0-1 背包

```
问题: 有 n 个物品，重量 w[i]，价值 v[i]，背包容量 C，
     求能装入的最大价值（每个物品只能选一次）。

状态定义:
dp[i][j] = 前 i 个物品，容量 j 时的最大价值

状态转移:
对于第 i 个物品，有两种选择:
1. 不选: dp[i][j] = dp[i-1][j]
2. 选:   dp[i][j] = dp[i-1][j-w[i]] + v[i]（如果 j ≥ w[i]）

dp[i][j] = max(dp[i-1][j], dp[i-1][j-w[i]] + v[i])

初始化:
dp[0][j] = 0（0个物品，价值为0）
dp[i][0] = 0（容量为0，价值为0）

示例:
物品: [(w=2,v=3), (w=3,v=4), (w=4,v=5)]
容量: C=5

dp 表格:
      容量 0  1  2  3  4  5
物品 0    0  0  0  0  0  0
物品 1    0  0  3  3  3  3   (选物品1)
物品 2    0  0  3  4  4  7   (选物品1+2)
物品 3    0  0  3  4  5  7   (选物品1+2最优)

结果: dp[3][5] = 7

空间优化:
使用一维数组，从后往前更新:
dp[j] = max(dp[j], dp[j-w[i]] + v[i])
```

## 4. 数据结构

```java
public class DynamicProgramming {
    
    // 爬楼梯
    public static int climbStairs(int n) {
        if (n <= 1) return 1;
        
        int[] dp = new int[n + 1];
        dp[0] = 1;
        dp[1] = 1;
        
        for (int i = 2; i <= n; i++) {
            dp[i] = dp[i - 1] + dp[i - 2];
        }
        
        return dp[n];
    }
    
    // 空间优化的爬楼梯
    public static int climbStairsOptimized(int n) {
        if (n <= 1) return 1;
        
        int prev2 = 1;  // dp[i-2]
        int prev1 = 1;  // dp[i-1]
        
        for (int i = 2; i <= n; i++) {
            int curr = prev1 + prev2;
            prev2 = prev1;
            prev1 = curr;
        }
        
        return prev1;
    }
    
    // 0-1 背包
    public static int knapsack(int[] w, int[] v, int C) {
        int n = w.length;
        int[] dp = new int[C + 1];
        
        for (int i = 0; i < n; i++) {
            // 从后往前更新，避免重复计算
            for (int j = C; j >= w[i]; j--) {
                dp[j] = Math.max(dp[j], dp[j - w[i]] + v[i]);
            }
        }
        
        return dp[C];
    }
}
```

## 5. 最佳实践

1. **先找状态转移方程**，这是 DP 的核心。  
2. **从简单情况入手**，推导出递推关系。  
3. **注意初始化条件**，避免边界错误。  
4. **空间优化技巧**：滚动数组、状态压缩。  
5. **画表格辅助理解**，特别是二维 DP。

### 常见 DP 类型

```
分类体系:

1. 线性 DP
   - 爬楼梯
   - 打家劫舍
   - 最长递增子序列

2. 区间 DP
   - 最长回文子串
   - 矩阵链乘法

3. 背包问题
   - 0-1 背包
   - 完全背包
   - 多重背包

4. 树形 DP
   - 二叉树最大路径和
   - 树的最大独立集

5. 状态压缩 DP
   - 旅行商问题
   - 棋盘覆盖

6. 数位 DP
   - 统计范围内满足条件的数字个数
```
